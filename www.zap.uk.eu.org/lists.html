Zap mailing lists

There are a range of mailing lists pertaining to Zap, which are listed below. They are run using GNU Mailman; you can subscribe via the web interface, or by sending an email to <listname>-request@zap.uk.eu.org with the text subscribe in either the subject line or body. The list server will send back a confirmation email which contains instructions on how to finalise your subscription.

The web interface also allows you to configure some aspects of your subscription, such as whether you receive the list as posts are sent to it, or as a regular digest. You can also access the list archives in this way.

Note that we recently changed the names of the lists. Where previously they would have been of the form beta@zap.uk.eu.org, they are now zap-beta@zap.uk.eu.org.

All Zap lists are 'open' except announce; that is, you don't have to be subscribed to the lists to post to them (this is particularly important for things like the bug list). announce is moderated; each message sent to it is checked first by one of the Zap development team. Please try to ensure that you send your message to the correct list (especially in the case of bugs, because they should get fixed faster if they get sent to the right place :-).

zap-buglist
This is the list where bug reports are discussed. Please do not send bug reports to this address; use bugs@zap.uk.eu.org instead.
Note that email to this list will have its Reply-To header set to the list. This is evil, and I shouldn't do it, but I discovered I more or less had to, because so many people were replying to the bug submission email address instead of the list. See Reply-To Munging Considered Harmful for an overview of why this makes me shiver uncontrollably. Ideally I'll write a small filter which will add Reply-To to emails coming in from the submission address only, so I can remove this.
Subscribe to zap-buglist
zap-beta
This is the discussion list for people participating in beta testing of Zap. (You can 'become' a beta tester simply by downloading the latest beta version and starting to use it.) Please do not send beta bug reports to this address; use betabugs@zap.uk.eu.org instead.
Subscribe to zap-beta
zap-features
A largely non-technical list for discussion of proposed new features.
Subscribe to zap-features
zap-devel
A largely technical list for discussion of alterations and additions
to the core of Zap. Subscribe to zap-devel
zap-technical
Other technical issues: this is where to ask questions such as "How do I write a mode/command to do ..."
Subscribe to zap-technical
zap-announce (moderated)
A list to which announcements of important changes is made. In general these announcements will also be posted to comp.sys.acorn.announce.
Subscribe to zap-announce

If you need to filter mail from these lists, the best way to do it is to filter on the List-Id header. This will appear something like below. (If you need it, we have information specific to using the Zap mailing lists with Acorn mail software.) 

List-Id: Technical discussion of Zap development. <zap-devel.zap.uk.eu.org>
